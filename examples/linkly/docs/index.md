# Linkly — Docs

A worked end-to-end Living Docs corpus for a fictional URL shortener. It exists so the
discipline can be *seen*, not just described, and so `living-docs check` has a real
bundle to run against. Read it top-to-bottom to watch one feature flow down the trail:
**constitution → PRD → ADR + BDR → issue → code**.

* [Constitution](constitution.md) - product scope, data model, non-negotiables (root of trace)

## Trail

* [PRDs](prd/) - what the system must do and why
* [ADRs](adr/) - how it is structured, and why
* [BDRs](bdr/) - what it must observably do (Given/When/Then)
* [Issues](issues/) - execution slices

## Reference

* [Context](context/) - domain vocabulary + glossary
* [Architecture](architecture/) - living Mermaid views
